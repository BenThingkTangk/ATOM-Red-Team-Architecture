How ATOM Detects, Analyzes & Stops Attacks

ATOM's autonomous pipeline moves from sensor-level anomaly detection to containment without human-in-the-loop delay. Below is the autonomous lifecycle for each major attack category, from first signal to full mitigation.

Autonomous Response Lifecycle — Every Attack Type
01
Sense
HumanOS edge agents continuously collect kernel-level telemetry (process trees, syscalls, network flows, file events, registry changes, API calls). On-device ML inference runs behavioral models locally at <5ms — no round-trip required. Akamai EdgeWorkers simultaneously monitor all HTTP/S traffic at the CDN edge for request-layer anomalies.
⏱ Latency: <5ms from event to local classification
02
Enrich
Suspicious events are forwarded (PQC-encrypted) to the nearest Akamai Distributed Compute POP. The Stream Normalizer maps events to OCSF/ECS schema. ⏱ Latency: 50-100ms enrichment pipeline at edge POP
03
Correlate
CTI Core runs graph-based correlation across all POPs simultaneously — identifying campaign clustering, multi-stage attack sequences, and cross-organizational patterns invisible to any single-tenant tool. ⏱ Latency: 200-500ms cross-tenant correlation
04
Contain
High-confidence detections trigger autonomous containment actions pushed back down through the edge POPs to agents: endpoint isolation, process kill, credential revocation, network segmentation tightening via Guardicore, WAF rule injection via App & API Protector, DNS sinkholing, firewall block list updates. SOAR playbooks execute automatically for known attack patterns.
⏱ Latency: <30 seconds from correlation to containment action deployed

Learn & Share

Anonymized campaign intelligence is shared with CERT partners via STIX 2.1 using differential privacy and ZK proofs. Models retrain continuously via federated learning without centralizing raw data.
⏱ Global intel propagation: <60 seconds to all federated agents
